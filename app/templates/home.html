Instant Messages

Online Database

The service provided by our program allows to send and receive messages stored in an online database. Each user must create an account and to log in it to use to the service.

Intuitive Design

The graphics is simple and intuitive such that even user without any technical knowledge can use the service.

Security requirements

The privacy of messages is preserved thanks to a not elementary encryption system used to store them.

Technologies

How the web works?

Our web page operations chain:

  • Create a user

    Two types of users could be created: Normal and Super user

    Each new user must compile a form including its first and last name, a username needed for the log-in and a valid password of at least 5 characters.

  • Send a message

    Each user must authenticate with the username and passwords previously created and only then it could send messages to other users.

  • Encrypt the message

    Each message is store in the online database directly encrypted with a key dependent from some information of the receiving user.

  • Decrypt the message

    When a user want to visualize the inbox content, it just needs to log-in and it already has the key to decrypt all its incoming messages.

  • Inbox sight

    In the end each authenticate user is able to see its inbox folder.
